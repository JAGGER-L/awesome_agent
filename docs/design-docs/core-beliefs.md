# Core Beliefs

- Reliability comes from constraints, state, tools, and evidence, not model
  confidence.
- The repository and persisted runtime state are the source of truth.
- Simple tasks should remain solo; teams exist only when complexity justifies
  coordination overhead.
- Teammates own durable responsibilities. Subagents own bounded tasks.
- Verification must be independent from implementation in team mode.
- Side effects require explicit capabilities, approval policies, and isolation.
- Raw history remains immutable; prompt context may be compressed.
- Observability is a product capability, not optional debugging output.
- Important architecture rules should become executable checks.
- Harness rules must remain small, current, and measurable.
