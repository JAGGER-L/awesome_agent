# Execution Plans

Use an execution plan for work that spans multiple files, systems, or sessions.

Active plans live in `docs/exec-plans/active/`. Completed plans move to
`docs/exec-plans/completed/`.

Every active plan records:

- objective and confirmed decisions
- scope and explicit non-goals
- milestones
- validation commands
- evidence
- risks and blockers
- current handoff and next action

Only one milestone is in progress at a time. Code existing without successful
validation is not sufficient to complete a milestone.
