# Project Governance

This directory stores durable repository-level governance such as technical
debt and quality policy. It must not contain development-agent session plans or
`awesome_agent` runtime task state.

Use this directory for committed long-term project facts. Local Codex execution
plans belong under `.codex/exec-plans/` and must remain uncommitted. Historical
`.codex` roadmap drafts are not authoritative.
